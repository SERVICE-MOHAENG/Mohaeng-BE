# Git 커밋 컨벤션

## 커밋 메세지 형식
```
{type} (#{issue}) : {한국어 설명}
```

## 커밋 타입
| 타입 | 설명 |
|------|------|
| `feat` | 새로운 기능 추가 |
| `fix` | 버그 수정 |
| `refactor` | 코드 리팩토링 (기능 변경 없음) |
| `docs` | 문서 수정 |
| `test` | 테스트 추가/수정 |
| `chore` | 빌드, 설정 등 기타 변경 |
| `style` | 코드 포맷팅, 세미콜론 누락 등 |

## 규칙
- 커밋 메세지는 **한국어**로 작성
- description(본문) 작성 금지
- **하나의 커밋에 여러 파일을 몰아넣지 않고 논리적 단위로 분리**
- Co-authored-by 등 자동 생성 표시 추가 금지
- 이슈 번호는 반드시 포함

## 예시
```
feat (#120) : 나라별 도시 조회 API 컨트롤러 추가
fix (#123) : Python 서버 URL trailing slash 제거
refactor (#75) : 장소 엔티티 PK를 UUID에서 placeId로 변경
```
